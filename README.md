<p>
  <img src="https://img.shields.io/badge/Rasterio-3776AB?style=flat-square&logoColor=white"/>

  <img src="https://img.shields.io/badge/SAR%2FOptical-7B3F61?style=flat-square&logoColor=white"/>
</p>

Satellite-based flood detection system using multi-spectral NDWI analysis on optical and SAR imagery with real-time disaster telemetry ingestion.

<details>
<summary><b>Architecture and Technical Depth</b></summary>
<br/>

| Component | Implementation |
|:--|:--|
| **Flood Detection** | 3-stage morphological filtering pipeline (NumPy/SciPy) producing pixel-level inundation ratios for quantitative damage assessment |
| **Telemetry Engine** | Fault-tolerant GDACS XML ingestion with `georss`-namespace coordinate extraction, 3-tier severity-weighted scoring |
| **Spatial Aggregation** | 5 to 10 degree geospatial grid bucketing for regional aggregation across 7 disaster categories |
| **API Layer** | 8-route FastAPI service exposing inundation ratios and severity scores |
| **Reliability** | Cache-based failover ensures zero downtime during feed outages |

</details>

**Published:** Lead Author — *International Journal on Science and Technology (IJSAT)*, Vol. 16, Issue 2, Apr to Jun 2025
